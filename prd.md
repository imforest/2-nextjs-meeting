## AI 기반 회의록 자동화 도구 제품 요구사항 문서(PRD)

**버전:** 1.0
**작성일:** 2025년 11월 11일
**작성자:** 김기범
**최종 수정일:** 2025년 11월 11일

---

### **1. 제품 개요**

#### **1.1. 목적: 이 제품이 해결하고자 하는 문제와 사용자에게 제공하는 가치**

많은 직장인과 학생들은 회의나 강의에 참여한 후, 논의된 내용을 정리하고 핵심 사항을 파악하는 데 많은 시간을 소모합니다. 특히, 장시간의 회의 내용을 다시 듣고 수동으로 회의록을 작성하는 과정은 비효율적이며, 중요한 결정 사항이나 아이디어를 놓칠 위험이 있습니다.

본 제품은 **AI 기술을 활용하여 음성 기록을 텍스트로 자동 변환하고, 핵심 내용을 요약하여 사용자의 시간과 노력을 획기적으로 절약**하는 것을 목표로 합니다. 사용자는 회의에만 집중하고, 회의록 작성 및 정리에 대한 부담에서 벗어나 생산성을 높일 수 있습니다. 이를 통해 사용자는 다음과 같은 가치를 얻을 수 있습니다.

- **시간 절약:** 회의록 작성에 소요되는 시간을 평균 80% 이상 단축합니다.
- **업무 효율성 증대:** 정리된 회의록을 바탕으로 빠르게 후속 업무를 처리하고, 중요한 의사결정에 집중할 수 있습니다.
- **정보 누락 방지:** 모든 발언이 텍스트로 기록되므로, 회의의 모든 내용을 놓치지 않고 다시 확인할 수 있습니다.
- **정보 접근성 향상:** 텍스트 검색을 통>해 특정 발언이나 주제를 쉽게 찾을 수 있습니다.

#### **1.2. 범위: 제품의 주요 기능 경계**

초기 버전(v1.0)은 다음과 같은 범위에 집중합니다.

- **플랫폼:** 주요 기능은 웹 브라우저를 통해 접근 가능한 **웹 기반 애플리케이션**으로 제공됩니다.
- **사용자:** 개인 사용자가 자신의 회의를 기록하고 관리하는 **단일 사용자(Single-user)** 환경을 지원합니다. 팀 공유나 협업 기능은 다음 버전에서 고려됩니다.
- **데이터 저장:** 모든 데이터(음성 파일, 텍스트, 메모 등)는 사용자의 **로컬 컴퓨터에 저장**하여 개인정보 보호와 데이터 소유권을 강화합니다.
- **핵심 기능:** 음성 녹음, 텍스트 변환, AI 기반 요약/회의록 생성, 관련 파일 업로드 및 메모 작성을 핵심 기능으로 제공합니다.

#### **1.3. 대상 사용자: 이 제품을 사용할 주요 사용자 그룹 및 그들의 니즈**

- **그룹 1: IT/기술 직군 종사자 (개발자, 기획자, PM)**

  - **니즈:** 기술 미팅, 스프린트 회의 등 전문 용어가 많이 사용되는 회의 내용을 정확하게 기록하고 싶어 합니다. 또한, 회의에서 도출된 액션 아이템을 명확히 추적하고 관리해야 할 필요가 있습니다. JIRA, Slack 등 다른 협업 도구와의 연동을 통해 업무 흐름을 개선하고자 하는 요구가 있습니다.

- **그룹 2: 컨설턴트 및 전문직**

  - **니즈:** 고객과의 미팅, 인터뷰 내용을 상세하게 기록하고, 핵심 인사이트를 빠르게 파악하여 보고서를 작성해야 합니다. 데이터의 정확성과 보안을 매우 중요하게 생각합니다.

- **그룹 3: 학생 및 연구원**
  - **니즈:** 강의, 세미나, 연구 미팅 내용을 놓치지 않고 기록하여 학습 및 연구 자료로 활용하고자 합니다. 방대한 양의 정보를 효율적으로 요약하고 정리하는 기능이 필요합니다.

### **2. 기능 명세**

#### **2.1. 음성 녹음 기능**

- **설명:** 사용자는 웹 브라우저를 통해 마이크로폰 입력을 받아 실시간으로 음성을 녹음할 수 있습니다. 녹음된 음성은 시각적인 파형(Waveform)으로 표시되어 녹음이 진행 중임을 직관적으로 인지할 수 있게 합니다.
- **요구사항:**
  - [REQ-001] 사용자는 '녹음 시작' 버튼을 클릭하여 녹음을 시작할 수 있어야 합니다.
  - [REQ-002] 사용자는 '일시 정지' 및 '다시 시작' 버튼을 통해 녹음을 잠시 멈추고 재개할 수 있어야 합니다.
  - [REQ-003] 사용자는 '녹음 종료' 버튼을 클릭하여 녹음을 완전히 마칠 수 있어야 합니다.
  - [REQ-004] 녹음이 진행되는 동안, 현재까지의 녹음 시간이 'HH:MM:SS' 형식으로 표시되어야 합니다.
  - [REQ-005] 녹음 종료 시, 녹음된 오디오 데이터는 브라우저 내에서 즉시 재생 가능해야 하며, 로컬 파일 시스템에 저장할 수 있어야 합니다.

#### **2.2. 메모 기능**

- **설명:** 사용자는 회의가 진행되는 동안 또는 종료된 후에, 중요한 생각이나 추가 정보를 텍스트 메모로 기록할 수 있습니다. 이 메모는 해당 회의 데이터와 함께 저장 및 관리됩니다.
- **요구사항:**
  - [REQ-006] 각 회의 상세 화면에는 다중 라인 텍스트 입력이 가능한 메모 영역이 있어야 합니다.
  - [REQ-007] 사용자가 입력한 메모는 별도의 저장 버튼 없이 자동으로 저장되어야 합니다 (Autosave).
  - [REQ-008] 메모 영역은 기본적인 텍스트 서식(굵게, 기울임, 밑줄, 목록)을 지원해야 합니다.
  - [REQ-009] 작성된 메모는 해당 회의 데이터와 연결되어 언제든지 조회 및 수정이 가능해야 합니다.

#### **2.3. 파일 업로드 기능**

- **설명:** 사용자는 회의와 관련된 문서(PDF, DOCX, 이미지 등)나 이미 녹음된 음성 파일(MP3, WAV 등)을 업로드하여 회의 데이터와 함께 관리할 수 있습니다.
- **요구사항:**
  - [REQ-010] 사용자는 '파일 업로드' 버튼 또는 드래그 앤 드롭(Drag-and-drop) 방식으로 파일을 첨부할 수 있어야 합니다.
  - [REQ-011] 한 번에 여러 개의 파일을 업로드할 수 있어야 합니다.
  - [REQ-012] 업로드된 파일 목록은 해당 회의 상세 페이지에 표시되며, 클릭 시 다운로드할 수 있어야 합니다.
  - [REQ-013] 업로드 가능한 파일의 최대 크기는 100MB로 제한합니다.
  - [REQ-014] 외부 음성 파일을 업로드할 경우, 해당 파일을 AI 기반 회의록/요약 생성 기능에 사용할 수 있어야 합니다.

#### **2.4. 회의 데이터 관리 기능**

- **설명:** 생성된 모든 회의 기록(음성, 텍스트, 메모, 첨부 파일 등)을 목록 형태로 관리하고, 사용자가 쉽게 검색하고 접근할 수 있도록 하는 기능입니다.
- **요구사항:**
  - [REQ-015] 메인 화면에는 모든 회의 기록이 최신순으로 정렬된 목록이 표시되어야 합니다.
  - [REQ-016] 각 목록 항목에는 회의 제목, 날짜, 시간, 요약의 첫 문장 등 핵심 정보가 표시되어야 합니다.
  - [REQ-017] 사용자는 회의 제목이나 내용의 키워드를 사용하여 원하는 회의 기록을 검색할 수 있어야 합니다.
  - [REQ-018] 사용자는 각 회의 기록의 제목을 수정하거나 기록 자체를 삭제할 수 있어야 합니다.

#### **2.5. AI 기반 회의록/요약 생성 기능**

- **설명:** 녹음된 음성 또는 업로드된 음성 파일을 AssemblyAI를 통해 텍스트로 변환하고, 변환된 전체 텍스트를 GPT 모델을 활용하여 핵심 요약본과 상세 회의록으로 자동 생성합니다.
- **요구사항:**
  - [REQ-019] 녹음 종료 또는 음성 파일 업로드 시, 'AI 회의록 생성' 버튼이 활성화되어야 합니다.
  - [REQ-020] 'AI 회의록 생성' 버튼을 클릭하면, 먼저 음성 파일이 AssemblyAI API로 전송되어 텍스트로 변환(STT: Speech-to-Text)되어야 합니다.
  - [REQ-021] 텍스트 변환이 완료되면, 해당 텍스트와 설정된 프롬프트를 조합하여 GPT API로 전송하고, '핵심 요약'과 '상세 회의록' 두 가지 결과물을 받아와야 합니다.
  - [REQ-022] 생성 과정 중에는 사용자에게 로딩 상태(예: 스피너, 진행률 표시)를 명확하게 보여주어야 합니다.
  - [REQ-023] 생성된 '핵심 요약'과 '상세 회의록'은 각각 별도의 탭이나 섹션에 명확히 구분되어 표시되어야 하며, 사용자는 생성된 텍스트를 복사하거나 편집할 수 있어야 합니다.

#### **2.6. 설정 관리 기능: AI 프롬프트 (요약용, 상세 회의록용)**

- **설명:** 사용자가 AI가 생성하는 결과물의 스타일과 형식을 직접 제어할 수 있도록 GPT에 전달될 프롬프트(Prompt)를 직접 수정하고 저장하는 기능입니다.
  - **프롬프트(Prompt)란?** AI 모델에게 원하는 결과물을 생성하도록 지시하는 명령어 또는 질문입니다.
- **요구사항:**
  - [REQ-024] 설정 화면(또는 팝업) 내에 'AI 프롬프트 설정' 메뉴가 있어야 합니다.
  - [REQ-025] '요약용 프롬프트'와 '상세 회의록용 프롬프트'를 입력할 수 있는 별도의 텍스트 입력 영역이 제공되어야 합니다.
  - [REQ-026] 각 프롬프트 입력 영역 하단에는 기본값으로 되돌릴 수 있는 '기본값 복원' 버튼이 있어야 합니다.
  - [REQ-027] 사용자가 수정한 프롬프트는 로컬에 저장되어 다음 생성 시에 자동으로 적용되어야 합니다.
  - **예시 프롬프트:**
    - **요약용:** "다음 회의 내용을 핵심 주제별로 3줄 요약해줘. 중요한 결정 사항은 꼭 포함해줘."
    - **상세 회의록용:** "다음 회의 내용을 바탕으로, 논의된 주요 안건, 각 안건에 대한 참석자들의 주요 발언, 결정된 사항, 그리고 향후 실행해야 할 액션 아이템 목록을 포함하는 상세 회의록을 작성해줘."

#### **2.7. 데이터 외부 연동 (웹훅) 기능**

- **설명:** 회의록 생성이 완료되었을 때, 지정된 URL로 회의록 데이터를 전송하여 다른 서비스(예: Slack, Notion, JIRA)와 연동할 수 있게 하는 기능입니다.
  - **웹훅(Webhook)이란?** 특정 이벤트가 발생했을 때, 한 서비스가 다른 서비스로 실시간 정보를 자동으로 전송하는 방법입니다.
- **요구사항:**
  - [REQ-028] 설정 화면에 '웹훅 URL'을 입력하고 저장할 수 있는 입력 필드가 있어야 합니다.
  - [REQ-029] 사용자는 웹훅 기능의 활성화/비활성화를 토글 스위치로 제어할 수 있어야 합니다.
  - [REQ-030] AI 회의록/요약 생성이 성공적으로 완료되면, 저장된 웹훅 URL로 회의 제목, 생성 시간, 요약 내용, 상세 회의록 본문을 포함하는 JSON 형식의 데이터를 HTTP POST 요청으로 전송해야 합니다.
  - [REQ-031] 웹훅 전송 성공 또는 실패 시, 사용자에게 알림(Toast message 등)을 통해 피드백을 제공해야 합니다.

### **3. UI/UX 디자인 고려사항**

#### **3.1. 주요 화면 레이아웃 구성**

- **헤더 (Header):**
  - 좌측: 서비스 로고
  - 우측: '새 회의 시작' 버튼, '설정' 아이콘
- **메인 콘텐츠 영역 (Main Content Area):**
  - **회의 목록 화면:** 좌측에는 회의 기록 목록이 표시되고, 우측에는 선택된 회의의 상세 내용(요약, 회의록, 메모 등)이 표시되는 2단 레이아웃을 사용합니다.
  - **회의 진행 화면:** 중앙에 녹음 상태(시간, 파형) 표시, 하단에 녹음 제어 버튼(시작, 일시정지, 종료), 우측에 실시간 메모 입력창을 배치합니다.

#### **3.2. 필요한 주요 UI 컴포넌트 목록**

- **버튼:** 녹음 시작/종료, 파일 업로드, AI 회의록 생성, 설정 저장 등
- **텍스트 표시 영역:** AI가 생성한 요약 및 상세 회의록 표시
- **텍스트 입력 필드:** 메모 입력, 회의 제목 수정, 프롬프트 및 웹훅 URL 설정
- **팝업/모달:** 설정 화면, 파일 업로드 창, 삭제 확인 창
- **아이콘:** 설정, 삭제, 복사, 다운로드 등 직관적인 아이콘 사용
- **로딩 인디케이터:** AI 생성, 파일 업로드 등 비동기 처리 시 사용자에게 대기 상태를 알려주는 스피너 또는 프로그레스 바
- **토글 스위치:** 웹훅 활성화/비활성화
- **탭:** 요약/상세 회의록/메모/첨부파일 등 콘텐츠 전환

#### **3.3. 일관된 디자인 시스템 적용**

- **컬러 팔레트:** 주 색상, 보조 색상, 텍스트 색상, 에러/성공 색상을 정의하여 일관되게 적용합니다.
- **타이포그래피:** 제목, 부제목, 본문 텍스트의 글꼴, 크기, 굵기를 정의하여 위계질서를 명확히 합니다.
- **컴포넌트 스타일:** 버튼, 입력창, 팝업 등 모든 UI 컴포넌트의 디자인(크기, 여백, 그림자 등)을 통일하여 일관된 사용자 경험을 제공합니다. (Tailwind CSS의 유틸리티 기반 접근법을 적극 활용)

### **4. 기술 스택**

- **프론트엔드 (Frontend):** **Next.js**
  - **선택 이유:** React 기반으로 뛰어난 개발 경험을 제공하며, 서버 사이드 렌더링(SSR) 및 정적 사이트 생성(SSG)을 지원하여 초기 로딩 속도와 검색 엔진 최적화(SEO)에 유리합니다. 파일 기반 라우팅 시스템은 직관적인 페이지 관리를 가능하게 합니다.
- **스타일링 (Styling):** **Tailwind CSS**
  - **선택 이유:** 유틸리티-우선(Utility-first) 접근법을 통해 미리 정의된 클래스를 조합하여 빠르고 일관된 UI 개발이 가능합니다. 디자인 시스템을 코드로 관리하기 용이하며, 커스터마이징이 자유롭습니다.
- **음성 -> 텍스트 변환 (Speech-to-Text):** **AssemblyAI**
  - **선택 이유:** 높은 정확도의 다국어 음성 인식 기능을 제공하며, 화자 분리(Speaker Diarization), 자동 챕터 생성 등 회의록 정리에 유용한 부가 기능을 API 형태로 쉽게 활용할 수 있습니다.
- **회의 요약/회의록 생성 (AI Summary):** **GPT (OpenAI API)**
  - **선택 이유:** 현재 가장 뛰어난 자연어 처리(NLP) 성능을 보여주는 언어 모델 중 하나로, 복잡한 회의 내용을 이해하고 사용자의 요구사항(프롬프트)에 맞는 자연스러운 요약 및 회의록을 생성하는 데 적합합니다.
- **데이터 저장 (Data Storage):** **로컬 저장소 (Local Storage / IndexedDB)**
  - **선택 이유:** 초기 버전에서는 별도의 서버 및 데이터베이스 구축 없이 사용자의 브라우저 내 저장소를 활용합니다. 이를 통해 개발 복잡도를 낮추고, 사용자의 데이터 프라이버시를 보장할 수 있습니다. IndexedDB는 대용량의 구조화된 데이터를 저장하는 데 적합하여 회의 데이터를 관리하기에 용이합니다.
